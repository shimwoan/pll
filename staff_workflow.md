# PLL 로펌 스태프 업무 워크플로우

> 스태프 관점에서 각 단계별 구체적인 행동 체크리스트

---

## 1단계. Intake / 사건수임

### 전화 상담 (미팅 전)
- [ ] 고객(또는 소개자) 전화 수신
- [ ] 고객 기본정보 수집
  - 의뢰인 이름, 연락처
  - 사고 일자 및 장소
  - 사건 유형 (교통사고 등)
  - 상대방 보험 유무
  - 치료 여부
- [ ] SOL(소멸시효) 확인
- [ ] 과실 여부 확인
- [ ] 보험 커버리지 가능성 검토
- [ ] 케이스 적합성 1차 판단
- [ ] 모호한 케이스는 대표님께 즉시 보고 및 수임여부 결정
- [ ] 미팅 약속 일시 및 장소 확정
- [ ] 퇴근 시간 이후 미팅이라면 초과근무 신청 및 승인

### 고객 대면 미팅 (현장 출장)
- [ ] 약속 장소로 출장 이동
- [ ] 고객 신원 확인
- [ ] 사건 상세정보 추가 파악
- [ ] 케이스 수임 최종 판단
- [ ] Retainer 서류 작성 및 전자서명 받기
- [ ] 고객으로부터 사고 사진 문자 수신 확인
- [ ] 기타 필요 서류 수집

### 회사 복귀 후
- [ ] Retainer 서류 Claim 부서에 이메일로 전달

---

## 2단계. Claim / 사고통지

- [ ] Intake에서 받은 Retainer 서류 확인
- [ ] 시스템에 사건 등록 (사건 생성)
- [ ] Retainer 서류 시스템에 업로드
- [ ] 케이스 매니저(CM) 배정
  - 언어(Spanish/Korean/English) 및 난이도(경력) 고려
  - 배정 완료 후 대표님/Office Manager cc 포함 이메일 회신
- [ ] 보험사에 당일 연락하여 Claim 오픈
- [ ] 보험사에 LOR(대리개시통지서) 발송
- [ ] 보험사로부터 Claim Number 수령 확인
- [ ] 보험사 담당자(Adjuster) 정보 수집 및 기록

---

## 3단계. Case Manager (CM) / 케이스 관리

### ① 클라이언트 초기 연락
- [ ] 고객에게 Thank You Letter 발송

### ② 예외 사항 체크
- [ ] DMV에 사건보고 및 자료 요청
  - SR-1, Prop 213, SRS, SR-19, Logan 조회 등
  - 해당 사항 태그 등으로 별도 표시
- [ ] 정부보험(Medicare/Medi-Cal) 여부 확인
  - 해당 시 Medical Collection 부서에 전달

### ③ 보험사 연락
- [ ] BI(대인) 클레임 번호 확보 및 담당자 연락
- [ ] PD(대물) 클레임 번호 확보 및 담당자 연락
- [ ] 양쪽 보험사에 클레임 서류 전달
- [ ] 커버리지 유무 확인 요청
- [ ] Liability(책임) 인정 여부 확인 요청

### ④ 보험사 1차 답변 검토
- [ ] Liability Letter / Coverage Letter 수신
- [ ] 커버리지 있음/없음 확인
- [ ] Liability 인정 여부 확인
  - 상대 책임 인정 → ⑤~⑥ 업무 진행
  - 우리 측 책임 있다 → Liability 다툼 진행
  - 최종 Deny → Drop 또는 Litigation 이관 검토

### ⑤ 병원에 의료기록 요청
- [ ] 관련 의료기관에 진료기록(Medical Records) 요청
- [ ] 의료비 청구서(Billing) 요청

### ⑥ 클라이언트 Follow-up

#### BI 클레임 (대인손해)
- [ ] 고객 치료 계획(Plan) 확인
- [ ] 치료 진행 상황 주기적 팔로우업
- [ ] 병원 의료기록 요청 → Medical Collection 부서에 전달
- [ ] 고객이 보험 한도 내 최대 치료를 받을 수 있도록 관리

#### PD 클레임 (대물손해)
- [ ] 차량 점검(Inspection) 일정 조율
- [ ] 차량 수리 진행 상황 체크
- [ ] Loss of Use(차량 미사용 손해) 보상 확인 (렌트카 지원 등)
- [ ] 차량 내 물품 확인 (카시트, 개인 소지품 등)

## 4단계. Medical Collection / 의료기록관리

### ① 의료기록 관리
- [ ] 진료기록(Medical Records) 수령 여부 확인
- [ ] 의료비 청구서(Billing) 수령 여부 확인
- [ ] 누락 항목 확인 및 의료기관 직접 연락
- [ ] CM의 정보 입력 누락/Form 오발송/서명 누락 체크

### ② 치료 타임라인 관리
- [ ] 치료 시작 여부 확인
- [ ] 치료 진행 상황 주기적 체크
- [ ] SOL 기한 관리
  - BI: 사고발생일로부터 2년
  - PD: 사고발생일로부터 3년

### ③ 정부보험 처리 (해당 케이스)
- [ ] 정부보험 여부 파악 (Medicare / Medi-Cal)
- [ ] 병원 발급 서류로 정부보험 상세 확인
- [ ] 정부보험 포털(예: Medi-Cal Portal)에서 상태 조회 및 답변 레터 수령
- [ ] 수령 서류 검토 (잘못 들어간 치료, 누락 치료 확인)

### ④ Demand 부서에 자료 전달
- [ ] 모든 의료 자료 수집 완료 확인
- [ ] Demand 부서에 의료기록 전달
- [ ] 예외 케이스 처리
  - 치료 장기화 시: 미처리업무 Task 생성 후 Demand로 단계 변경
  - 정부보험 케이스: 미처리업무 Task 생성 후 Demand로 단계 변경

---

## 5단계. Demand / 보상요구

### ① 자료 검토 및 서류 작성
- [ ] Medical Collection에서 수령한 의료기록, 빌링, 치료내용, 사고자료 확인
- [ ] 향후 치료비(Future Treatment) 계산
  - 현재까지 발생한 의료비(Medical Bills) 집계
  - 앞으로 예상되는 치료비 합산
  - 의사 소견 기반 향후 치료 가능성 주장 근거 정리

### ② 변호사 리뷰
- [ ] Demand 서류 변호사님께 제출
- [ ] 변호사 검토 완료 확인 (발송 전 반드시 거쳐야 하는 Final Check)

### ③ Demand 발송
- [ ] 상대측 보험사에 Demand Package 발송
- [ ] 발송 일자 기록

---

## 6단계. Negotiation / 협상

### ① ACK 확인 및 리마인더
- [ ] Demand 발송일 기록
- [ ] 30일 이내 보험사 ACK(접수확인서) 수신 여부 모니터링
- [ ] 30일 내 미응답 시 Acknowledge 리마인더 발송

### ② 보험사 답변 수신 및 처리
- [ ] 보험사 회신 수신 (이메일 및 우편 모두 확인)
- [ ] 우편 수신 시 스캔하여 시스템에 업로드
- [ ] 중복 회신(이메일+우편) 여부 확인
- [ ] 회신 유형 확인 및 처리:
  - **Settlement Offer**: 제안 합의금 확인 → 검토 후 Counter Demand 준비
  - **Rejection/Denial Letter**: 거절 이유 확인 → 자료 보완 또는 Litigation 검토
  - **Request for Information**: 추가 자료 준비 및 발송
    - 요청 자료: Statement, Video, Doctor Note, EOB, 건강보험 미커버 레터 등
  - **Policy Limit Tender**: 보험 한도 전액 제시 확인 → 수락 여부 검토
  - **Global Tender**: 전원 일괄 합의 제안 → 금액 분배 협의

### ③ 보험사 제안 검토
- [ ] Rejection 이유 분석
- [ ] Policy Limit 확인
- [ ] 자료 보완 필요 여부 판단

### ④ 협상 진행
- [ ] Counter Demand 작성 및 발송 (Policy Limit 미달 시)
- [ ] 보험사 Counter Offer 수신 및 검토
- [ ] 최종 합의 도달까지 반복 협상

---

## 7단계. Settle / 합의

### ① 합의 금액 확정
- [ ] 보험사와 최종 합의 금액 확정

### ② 고객 동의서 수령
- [ ] 고객에게 Client Release(동의서) 발송
- [ ] 고객 서명 완료 확인

### ③ 보험사에 Release 전달
- [ ] 서명 완료된 Release 보험사에 전달
- [ ] 보험사 ACK 수신 확인

### ④ 내부 이관
- [ ] 시스템에서 담당자 Accounting 부서로 변경
- [ ] Teams 메신저로 Release 발송 완료 Accounting 부서에 알림

### 예외 처리

#### Petition for Minor's Compromise (미성년자 포함 케이스)
- [ ] 미성년자 피해자 여부 확인 및 태그 표시
- [ ] 보험사와 합의금 결정
- [ ] 미성년자 부모에게 Release 서명 수령
- [ ] Petition for Minor's Compromise 서류 제출 (Litigation 부서 담당)
- [ ] OSC(Order to Show Cause) 헤어링 일정 확인 (5,000불 이상 시)
- [ ] 변호사님 헤어링 참석 일정 조율
- [ ] 법원 승인(Court Approval) 수령
- [ ] Court Approval 보험사에 발송
- [ ] 보험사 수표(Check) 수령
- [ ] 은행 동행하여 Block Account 개설 및 입금 확인

#### UIM Claim (보험 한도 부족 케이스)
- [ ] UIM 조건 충족 여부 확인 및 태그 표시
  - 고객 보험한도 > 상대방 보험한도
  - 상대방 최대 보험한도로 합의 완료
  - 총 손해액 > 수령한 합의금
- [ ] 고객 보험사에 UIM Demand 발송
- [ ] 보험사 ACK 30일 이내 수신 모니터링
- [ ] 보험사 회신 확인 및 검토
- [ ] Release 담당자 배정
- [ ] Release 발송 → 고객 서명 → 보험사 전달
- [ ] Lien(Accounting) 부서로 이관

## 8단계. Accounting / Lien / 회계

### ① 합의금 수령
- [ ] 보험사로부터 Settlement Check 우편 수신
- [ ] 합의금 Trust Account에 보관

### ② 병원 의료비 협상
- [ ] 각 병원과 의료비 감액 협상 진행
- [ ] 협상 전/후 금액 기록
  - Original Amount, Reduced Amount, Reduction Difference
- [ ] 정부보험(Medicare/Medi-Cal)과 비용 절감 협상

### ③ 비용 공제 처리
- [ ] Legal Fee 정산
  - 변호사 수임료 (합의금의 33%)
  - 정부/공공기록 관련 비용 (예: 사고보고서)
- [ ] 소송 경비(Case Expenses) 정산
  - Filing Fee, Translator, Messenger, 기타 행정비용
- [ ] Lien 비용 정산
  - 의료비, 정부보험사, 기타 Lien (양육비, 세금체납 등)
- [ ] 고객이탈방지 기록 확인
  - Attorney fee 할인 내역
  - Advancement(선지급) 내역
  - Legal Funding 내역

### ④ 고객 정산서 발송
- [ ] Disbursement Sheet(정산서) 작성
  - 공제금 차감 내역, 환급금 포함
- [ ] 정산서 고객에게 발송
- [ ] 고객 최종 동의 서명 수령

### ⑤ 지급 처리
- [ ] 병원에 Check 발송
- [ ] 고객에게 Check 발송
- [ ] Check Cleared(실제 결제 완료) 은행에서 확인

### Sub-out (사건 이관) 시 추가 업무
- [ ] Itemized Bill(정산서) 작성
- [ ] Itemized Bill 상대 로펌에 발송
- [ ] 발송 누락 여부 최종 확인

## 9단계. Litigation / 소송 (합의 실패 시)

- [ ] Pre-suit 평가: 소송 필요성 판단
- [ ] 소장(Complaint/Petition) 작성
- [ ] 법원에 소장 접수
  - Small Claims (≤ $12,500)
  - Limited Civil (≤ $25,000)
  - Unlimited Civil (> $25,000)
- [ ] 소장 송달(Service) 진행
- [ ] 송달 후 30일 이내 Answer 제출
- [ ] Discovery 일정 관리 및 마감일 내 처리
  - 문서 요청, 진술서, 증인 목록, Interrogatories(서면질문서)
- [ ] 선서/증언녹취(Deposition) 일정 관리
- [ ] 의료기록 요약(Medical Summary) 작성
- [ ] 손해액(Damages) 계산
- [ ] 조정/화해(Mediation) 준비

---

## 언제든 발생 가능한 업무

### Sub-out / 사건 이관
- [ ] 이관 결정 확인
- [ ] Itemized Bill 작성 및 발송 (누락 시 미수금 발생)
- [ ] 사건 관련 서류 정리 및 이관

### Dropped / 케이스 취하
- [ ] 취하 결정 확인
- [ ] 사건 종결 처리

---

## 사건 종결 (Closed)

- [ ] 모든 비용 정산 완료 확인
- [ ] 고객 최종 지급 완료 확인
- [ ] Check Cleared 최종 확인
- [ ] 사건 시스템에서 Closed 처리
